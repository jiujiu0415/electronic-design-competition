### 1.2 从 AD9833 原理图里看出了什么

```
AD9833 V40 模块原理图 → 关键发现：

① 主芯片：AD9833BRMZ（10脚 MSOP 封装）
② 晶振：25MHz 有源晶振（决定了频率精度）
   频率分辨率 = 25MHz / 2^28 ≈ 0.093 Hz
③ 没有后级运放！VOUT 直接从芯片出来
   输出 ≈ 0.6Vpp（峰峰值很小）
④ 引脚：FSYNC、SCLK、SDATA、VCC、GND、VOUT
⑤ 3.3V 供电
```

```
STM32F103C8T6 Blue Pill → 关键发现：

① 外部晶振：8MHz（HSE）
② GPIO 全部引出到排针
③ PA4/PA5/PA7 可用于 SPI1 + 软件片选
④ PA9/PA10 可用于 USART1 串口
⑤ 3.3V 供电
```

## <a id="二"></a>阶段二：理解通信方式 — SPI 到底是什么

### 2.1 STM32 和 AD9833 之间怎么"说话"？

AD9833 用的是一种叫 **SPI（串行外设接口）** 的通信协议。

最简单的类比：
```
老师（STM32）在课堂上点名让学生（AD9833）回答问题：

① 喊名字（拉低 FSYNC）  → "AD9833, 听好了"
② 念题目（MOSI 发命令） → "把频率调到 1kHz"
③ 记答案（不需要，AD9833 只听话不回答）
④ 让他坐下（拉高 FSYNC） → "可以了, 执行"
```

### 2.2 4 根线的分工

| 线 | 方向 | 比喻 |
|----|------|------|
| **SCLK** (时钟) | STM32→AD9833 | 节拍器，告诉对方"现在读" |
| **MOSI** (主出从入) | STM32→AD9833 | 老师念的题目 |
| **MISO** (主入从出) | AD9833→STM32 | 学生回答（本项目不接，AD9833 只收不发） |
| **FSYNC** (片选) | STM32→AD9833 | 点名，低电平=被选中 |

### 2.3 四种 SPI 模式的秘密 —— AD9833 用哪种？

SPI 有 4 种"节奏"，由两个参数决定：

```
CPOL（时钟空闲时是高还是低？）
CPHA（在第几个边沿采样数据？）
```

查看 AD9833 手册第 7 页的 **时序图**：

```
SCLK 空闲时 = 高电平  → CPOL = 1
数据在下降沿被采样    → CPHA = 0（第一个边沿 = 第一个下降沿）
```

**结论：Mode 2 (CPOL=1, CPHA=0)**

### 2.4 AD9833 一个最特殊的地方 —— FSYNC 必须每 16-bit 拉高一次

```
普通 SPI 设备：CS 拉低后可以连续发几百字节

AD9833：
  FSYNC ──┐  ┌──┐  ┌──┐  ┌──
           └──┘  └──┘  └──┘
          ←16bit→←16bit→←16bit→
  
  每发 16 位必须拉高一次！这是 AD9833 的命令帧格式。
```

这就是为什么我们要 **软件控制 FSYNC**（用 PA4 做 GPIO），而不是用 STM32 的硬件 NSS。

## <a id="三"></a>阶段三：对照手册配置芯片 — CubeMX 每一步为什么这么配

### 3.1 时钟配置

```
HSE: Crystal/Ceramic Resonator (用外部 8MHz 晶振)
PLL Source: HSE
PLL Mul: x9 → 8MHz × 9 = 72MHz
System Clock: PLLCLK (72MHz)
APB1: /2 = 36MHz
APB2: /1 = 72MHz  （SPI1 在 APB2 上，需要高速）

手册依据：STM32F103 最高 72MHz
```

### 3.2 SPI1 配置（每一项都能在手册里找到依据）

| 配置项 | 值 | 手册依据 | 为什么 |
|--------|-----|---------|--------|
| Mode | Full-Duplex Master | — | STM32 是主控 |
| **Data Size** | **16 Bit** | 第6页 "每个写周期=16位串行数据" | AD9833 命令固定 16 位 |
| **Baud Rate** | **256 (≈281kHz)** | 第6页 "SCLK max=40MHz" | 281kHz << 40MHz，稳 |
| **CPOL** | **High** | 第7页时序图: 空闲=高 | 手册画着空闲时 SCLK 是高 |
| **CPHA** | **2 Edge** | 第7页时序图: 下降沿锁存 | CPHA=2Edge + CPOL=High = Mode2 |
| **NSS** | **Software** | 第7页 FSYNC 时序 | 硬件 NSS 做不了每16bit拉高 |
| **First Bit** | **MSB First** | 第7页: D15在前 | 高位先发 |
| **FSYNC(PA4)** | **GPIO Output** | 第7页 FSYNC 时序 | 手动控制帧边界 |

### 3.3 USART1 配置（用于串口命令控制）

```
Baud Rate: 115200 (够快)
Word Length: 8 Bits (标准)
Parity: None
Stop Bits: 1
NVIC: USART1 global interrupt ✓ (收到字符时自动进中断)
```

## <a id="四"></a>阶段四：从数据手册到代码 — 每一个宏定义都能在手册里找到

### 4.1 代码的底层逻辑

整个驱动的设计思路是"搭积木"：

```
手册的每一位 → 一个 #define 积木块
波形模式      → 把需要的积木块 OR 在一起
函数          → 把拼好的积木通过 SPI 发出去
```

### 4.2 ad9833.h — 积木块清单

#### 第一组积木：命令类型（D15-D14）
```
手册第 8 页表 8：

00 → CMD_WRITE_CTRL  (控制寄存器)
01 → CMD_WRITE_FREQ0 (频率寄存器0)
10 → CMD_WRITE_FREQ1 (频率寄存器1)
11 → CMD_WRITE_PHASE0 (相位寄存器0)
```

#### 第二组积木：控制寄存器的每一位（D13-D0）
```
手册第 8-9 页每一位的说明：
                                           手册位置
B28     (1<<13) → 连续28bit写频率           第8页 D13
HLB     (1<<12) → 分高低14位写               第8页 D12
FSELECT (1<<11) → 切换FREQ0/FREQ1           第8页 D11
PSELECT (1<<10) → 切换PHASE0/PHASE1         第8页 D10
PIN_SW  (1<<9)  → 软件/硬件控制             第8页 D9
RESET   (1<<8)  → 复位（配频率时要置1）      第8页 D8
SLEEP1  (1<<7)  → 关闭内部时钟              第8页 D7
SLEEP12 (1<<6)  → 关闭DAC                   第8页 D6
OPBITEN (1<<5)  → 方波输出模式              第9页 D5
SIGN_PIB(1<<4)  → 比较器模式                第9页 D4
DIV2    (1<<3)  → 方波二分频                第9页 D3
MODE    (1<<1)  → 正弦(0)/三角(1)           第9页 D1
```

#### 第三组积木：波形组合
```c
// 用 | 把需要的 bit 拼在一起：
CTRL_SINE     = BIT_RESET | BIT_B28                    // 正弦: 复位+B28
CTRL_TRIANGLE = BIT_RESET | BIT_B28 | BIT_MODE         // 三角: 复位+B28+MODE
CTRL_SQUARE   = BIT_RESET | BIT_B28 | BIT_OPBITEN | BIT_DIV2  // 方波: +OPBITEN+DIV2
```

### 4.3 ad9833.c — 怎么把积木发出去

#### 最底层：AD9833_WriteRegister()
```c
① 拉低 FSYNC       → 手册第7页 "FSYNC下降沿开始传输"
② HAL_SPI_Transmit → 发 16-bit（SPI 配置为 16-bit 模式）
③ 拉高 FSYNC       → 手册第7页 "FSYNC上升沿锁存数据"
```

#### 设频率：AD9833_SetFreq()
```c
① 频率字 = f_out × 2^28 / 25MHz    → 手册第8页公式
② 拆成低14位 + 高14位
③ 连续发两次：CMD_FREQ0 | 低14位,  CMD_FREQ0 | 高14位
   → 手册第8页 "B28模式下连续两次写入自动拼成28位"
```

#### 设波形：AD9833_SetWave()
```c
① 发波形控制字 (如 CTRL_SQUARE)    → 含 RESET=1
② 发 BIT_B28                      → RESET=0, 波形开始输出
   → 先复位再释放，避免切换时产生毛刺
```

## <a id="五"></a>阶段五：串口命令控制 — 让信号源"听话"

### 5.1 为什么加串口

裸的 AD9833 烧进去只能输出固定的波形。加串口后可以：

```
freq 1000     → 立刻切换到 1kHz
wave 1        → 立刻切换到三角波
sweep 100 10000 100 50 → 从100Hz扫到10kHz
status        → 查看当前状态
```

### 5.2 实现方式

```
中断接收模式（不在主循环里死等）：

串口收到一个字符 → 硬件自动触发中断
                → HAL_UART_RxCpltCallback()
                → 存入 rx_buf[]
                → 收到 \r\n 时置 rx_ready=1

主循环检测 rx_ready：
                → Cmd_Process(rx_buf)     // 解析 "freq 1000"
                → AD9833_SetFreq(1000)    // 调频率
                → 清空缓冲区，准备接收下一条
```

## <a id="七"></a>经验总结

### 7.1 这个项目的核心方法论

```
               ┌──────────────────┐
               │   数据手册 (DS)   │  ← 一切的根本
               │   第6-9页        │
               └───────┬──────────┘
                       │
          ┌────────────┼────────────┐
          │            │            │
     ┌────▼────┐ ┌────▼────┐ ┌────▼────┐
     │ CubeMX  │ │ 宏定义  │ │ 函数    │
     │ 配置    │ │ #define │ │ 实现    │
     │         │ │         │ │         │
     │ 手册的  │ │ 手册的  │ │ 手册的  │
     │ 时序→   │ │ 每一位→ │ │ 流程→   │
     │ 参数    │ │ 宏      │ │ 代码    │
     └─────────┘ └─────────┘ └─────────┘
```

**核心原则：写代码之前，先在手册里找到对应的描述。没有手册就没有代码。**

### 7.2 对于电赛备赛的启示

1. **拿到任何芯片，先看数据手册的通信时序图**——它告诉你 SPI 模式、数据格式、时序要求
2. **控制寄存器的每一 bit 都要理解**——它们是软件控制硬件的开关
3. **CubeMX 只是一个翻译器**——它把"CPOL=High, CPHA=2Edge"翻译成寄存器配置，但**为什么**这样配，只能从数据手册里找答案
4. **代码量其实很小**——ad9833.h 约 50 行有效代码，ad9833.c 约 80 行，main.c 约 120 行。复杂的是理解，不是代码量。
