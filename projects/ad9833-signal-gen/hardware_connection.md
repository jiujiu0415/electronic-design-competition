# AD9833 信号源 — 杜邦线连接指南

## 所需材料

| 物品 | 数量 |
|------|------|
| STM32F103C8T6 核心板 (Blue Pill) | 1 |
| AD9833 V40 模块 | 1 |
| 母对母杜邦线 | 5 根 |
| USB-TTL 串口模块 (CH340等) | 1 (调试用) |
| 示波器 (或逻辑分析仪) | 1 (看波形) |

## 连接总表

一共只需 **5 根线** 连接 STM32 ↔ AD9833：

```
STM32F103C8T6          AD9833 V40 模块
┌─────────────────┐    ┌──────────────────┐
│                 │    │                  │
│  PA4  ──────────┼────┼──▶ FSYNC         │
│  PA5  ──────────┼────┼──▶ SCLK          │
│  PA7  ──────────┼────┼──▶ SDATA         │
│  3.3V ──────────┼────┼──▶ VCC           │
│  GND  ──────────┼────┼──▶ GND           │
│                 │    │                  │
│                 │    │  VOUT ──▶ 示波器  │
└─────────────────┘    └──────────────────┘
```

## 逐根线说明

### 第 1 根：PA4 → FSYNC（黄色杜邦线）
| STM32 端 | AD9833 端 |
|----------|-----------|
| PA4 | FSYNC |

- **作用**：片选信号。STM32 每次要发命令时先拉低，发完拉高
- **为什么用 PA4**：SPI1 的硬件 NSS 自动片选满足不了 AD9833 "每 16bit 拉高一次" 的要求，所以用普通 GPIO 软件控制

### 第 2 根：PA5 → SCLK（绿色杜邦线）
| STM32 端 | AD9833 端 |
|----------|-----------|
| PA5 (SPI1_SCK) | SCLK |

- **作用**：SPI 时钟线，STM32 输出节拍信号
- **频率**：72MHz ÷ 256 = 281kHz（CubeMX 中配的）

### 第 3 根：PA7 → SDATA（蓝色杜邦线）
| STM32 端 | AD9833 端 |
|----------|-----------|
| PA7 (SPI1_MOSI) | SDATA |

- **作用**：SPI 数据线，STM32 通过这根线把命令发给 AD9833
- **注意**：AD9833 是"只写"设备，所以 MISO 线不用接

### 第 4 根：3.3V → VCC（红色杜邦线）
| STM32 端 | AD9833 端 |
|----------|-----------|
| 3.3V | VCC |

- **作用**：给 AD9833 供电
- **注意**：AD9833 工作电压 2.3V~5.5V，但模块板载 25MHz 有源晶振可能需要 3.3V。如果 5V 供电，先确认模块上的 LDO

### 第 5 根：GND → GND（黑色杜邦线）
| STM32 端 | AD9833 端 |
|----------|-----------|
| GND | GND |

- **作用**：共地！**这条是必须的**，没有共地，信号线上的电平对方读不懂
- **注意**：AD9833 模块上的 AGND（模拟地）、DGND（数字地）都接到这个 GND

## 上电前检查清单

- [ ] PA4↔FSYNC 连通？
- [ ] PA5↔SCLK 连通？
- [ ] PA7↔SDATA 连通？
- [ ] 3.3V↔VCC 连通？
- [ ] GND↔GND 连通？
- [ ] 每个杜邦线两头都插紧？
- [ ] 示波器探头接 VOUT？（探头地夹子接 GND）
- [ ] 用万用表测 VCC-GND 之间没有短路？

## 上电后预期现象

1. STM32 上电 → 板载 LED 亮
2. 串口助手 (115200 8N1) 显示欢迎信息
3. 示波器显示 **1kHz 正弦波**（约 0.6Vpp）
4. 输入 `freq 5000` → 波形变为 5kHz
5. 输入 `wave 1` → 波形变为三角波
6. 输入 `wave 2` → 波形变为方波
