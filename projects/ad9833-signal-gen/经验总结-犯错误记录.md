# AD9833 项目 — 犯错误经验总结

> 两个 bug，都是看似配置/代码"看起来没问题"但实际不符合数据手册。

---

## Bug #1：SPI 模式配错 — CPHA 选错了边沿

### 错误
CubeMX 中 CPHA 选了 **2 Edge**（= CPHA=1 = Mode 3）。

### 为什么没发现
`SPI_PHASE_2EDGE` 这个名字太像"第二个边沿"了，当时想当然觉得"下降沿 = 第二个边沿"，其实不是——命名里的 "2 Edge" 是指"第二个边沿是采样沿"，而 AD9833 需要第一个边沿采样。

### 手册真相（第 7 页时序图）
```
"Data is clocked into the AD9833 on the falling edge of SCLK"
时钟空闲 = 高电平 → CPOL = 1
下降沿采样    → 这是第 1 个边沿 → CPHA = 0
所以是 Mode 2 (CPOL=1, CPHA=0)
CubeMX 对应: CPOL=High, CPHA=1 Edge
```

### 教训
> **配任何一个芯片的 SPI 之前，必须先盯着时序图画出时钟空闲电平 + 哪个边沿采样，然后对照 CubeMX/HAL 的 CPOL/CPHA 命名去匹配。不要靠猜、不要靠"看起来差不多"。**

### 口诀
```
CPOL = 空闲电平 (0=低, 1=高)
CPHA = 在第几个边沿采样 (0=第1个, 1=第2个)

下降沿采样 + 空闲高 → CPOL=1, CPHA=0 → Mode 2
```

## Bug #2：释放复位时清掉了波形配置位

### 错误
```c
// 错误写法
AD9833_WriteRegister(CTRL_SQUARE);   // 设置了波形位 (OPBITEN=1, DIV2=1)
AD9833_WriteRegister(BIT_B28);       // 写了一个全新的控制字 → 波形位全变 0！
```

### 为什么没发现
正弦波的波形位恰好全是 0：

| 波形 | 需要的位 | BIT_B28 巧合？ |
|------|---------|---------------|
| 正弦 | (无额外位) | ✅ BIT_B28 = 0x2000, 波形位都是 0 |
| 三角 | MODE=1 | ❌ BIT_B28 把 MODE 清成 0 → 变回正弦 |
| 方波 | OPBITEN=1, DIV2=1 | ❌ BIT_B28 把两个都清 0 → 变回正弦 |

因为调试时先测试了正弦波（"能用"），就假设三角和方波也能用。**但正弦能用只是个巧合。**

### 正确写法
```c
AD9833_WriteRegister(wave_ctrl);                // 设波形 (含 RESET)
AD9833_WriteRegister(wave_ctrl & ~BIT_RESET);   // 只翻 RESET 那一位，其余不动
```

### 教训
> **用位掩码修改寄存器的一个 bit 时，必须用 `& ~BIT` 清零 + `| BIT` 置位，不能直接写一个新的"看起来差不多的值"。验证时要测试所有状态组合，不要因为一种情况 pass 就假设全对。**

### 口诀
```
想改某一个 bit → reg = (reg & ~BIT) | (要设的值)
想保持不变    → 不要写新的，在原来的基础上改
```
